Verify the rollout:

  kubectl -n {{ .Release.Namespace }} rollout status deployment/{{ include "argos-vm-collector.fullname" . }}
  kubectl -n {{ .Release.Namespace }} logs deployment/{{ include "argos-vm-collector.fullname" . }} --tail=50

The cloud account row appears in argosd within one polling interval. If
the row was not pre-created, the collector follows the hybrid-onboarding
flow (ADR-0015 §6) — register placeholder → admin pastes AK/SK in the UI
→ collector picks them up on the next refresh tick (every {{ .Values.credentialRefresh }}).

  curl -H "Authorization: Bearer <admin-pat>" \
    {{ .Values.serverURL }}/v1/cloud-accounts | jq '.items[] | select(.name=="{{ .Values.account.name }}")'
