# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Argos is a CMDB (Configuration Management Database) for Kubernetes environments, aligned with the ANSSI **SecNumCloud (SNC)** qualification framework. See `docs/adr/adr-0001-cmdb-for-snc-using-kube.md` for the foundational architectural decision.

## Stack

- **Language**: Go (1.23+)
- **Database**: PostgreSQL (with JSONB for heterogeneous Kubernetes specs)
- **API**: REST, contract-first via OpenAPI 3 (spec will live under `api/openapi/`)
- **Ingestion**: polling-based collector querying the Kubernetes API

## Layout

- `cmd/argosd/` — main entry point for the Argos daemon
- `internal/` — application packages (not importable externally); created as subsystems land
- `api/openapi/openapi.yaml` — OpenAPI 3.1 specification (contract source of truth)
- `migrations/` — PostgreSQL schema migrations (to be added)
- `docs/adr/` — Architectural Decision Records

## Architecture notes

The codebase currently covers the API layer only:

- `cmd/argosd/main.go` — daemon entry point: env-based configuration (`ARGOS_ADDR`, `ARGOS_DATABASE_URL`, `ARGOS_AUTO_MIGRATE`, `ARGOS_SHUTDOWN_TIMEOUT`, `ARGOS_BOOTSTRAP_ADMIN_PASSWORD`, `ARGOS_SESSION_SECURE_COOKIE`, `ARGOS_PUBLIC_LISTEN_TLS_CERT`, `ARGOS_PUBLIC_LISTEN_TLS_KEY`, `ARGOS_TRUSTED_PROXIES`, `ARGOS_REQUIRE_HTTPS`, `ARGOS_OIDC_*`, collector vars). Opens the PostgreSQL pool, runs migrations, bootstraps the first admin user when the DB has none, resolves the OIDC issuer's discovery document when configured (fatal on failure — operator must fix), starts the HTTP server with the dual-path auth middleware (cookie or bearer), spawns the collector goroutine when enabled, handles graceful shutdown on SIGINT / SIGTERM. Refuses to start if the removed `ARGOS_API_TOKEN` / `ARGOS_API_TOKENS` env vars are set — migrate to admin-panel-issued tokens per ADR-0007.

### Auth (ADR-0007)

Dual-path authentication resolved by `internal/auth.Middleware`:

- **Humans** log in either with username + password (local users) against `POST /v1/auth/login`, **or** via OIDC — `GET /v1/auth/oidc/authorize` 302s to the IdP, `GET /v1/auth/oidc/callback` exchanges the code + verifies the ID token's nonce + finds-or-creates a shadow user keyed on `(issuer, sub)`. Both paths mint the same server-side session cookie (HttpOnly, SameSite=Strict, 8h sliding expiry). Shadow users get role `viewer` on first login (admins promote manually — we don't trust OIDC group claims for authorization) and carry an unusable password hash so the local-login path refuses them.
- **Machines** use `Authorization: Bearer argos_pat_<prefix>_<suffix>`. Tokens are argon2id-hashed at rest with an 8-char plaintext prefix stored separately for O(1) lookup; created in the admin UI by an admin only; plaintext shown once and never again (GitHub-PAT pattern).

Four fixed roles mapped to scopes (plus a fifth machine-only token preset from ADR-0015):

| Role / preset   | Scopes                                                                         |
|-----------------|--------------------------------------------------------------------------------|
| `admin`         | `read` + `write` + `delete` + `admin` + `audit` (implies every endpoint **except `vm-collector`**) |
| `editor`        | `read` + `write`                                                               |
| `auditor`       | `read` + `audit`                                                               |
| `viewer`        | `read`                                                                         |
| `vm-collector`  | `vm-collector` only (token preset, never a user role; bound to a `cloud_account` UUID at issuance) |

The scope check in the OpenAPI `security:` blocks is unchanged from the pre-ADR-0007 world; both cookie and bearer paths feed the same downstream `caller{id, kind, role, scopes}` context. `auth.HasScope` short-circuits "admin implies everything" except for `vm-collector`, which only collector tokens can carry — see ADR-0015 §5.

First-run bootstrap creates a single admin user when `COUNT(users WHERE role='admin' AND disabled_at IS NULL) = 0`. Password comes from `ARGOS_BOOTSTRAP_ADMIN_PASSWORD` if set, else a random 16-char string printed **once** to the startup log inside a loud banner; `must_change_password=true` blocks every endpoint except `/v1/auth/change-password` until rotated. `/healthz` and `/readyz` stay unauthenticated.

OIDC is optional and enabled by setting `ARGOS_OIDC_ISSUER` (empty = off). When set, `ARGOS_OIDC_CLIENT_ID` and `ARGOS_OIDC_REDIRECT_URL` are required; `ARGOS_OIDC_CLIENT_SECRET` is required for confidential clients; `ARGOS_OIDC_SCOPES` (comma-separated, default `openid,email,profile`) and `ARGOS_OIDC_LABEL` (button text, default `OIDC`) are optional. The authorize-code flow carries PKCE (S256) + nonce + state; the in-flight state lives in `oidc_auth_states` as a one-shot row (SELECT FOR UPDATE + DELETE under `ConsumeOidcAuthState`). `GET /v1/auth/config` is public and tells the UI whether to render the "Sign in with <label>" button.

### Last-admin guard (ADR-0017 / AUTHZ-VULN-01/02)

`PATCH /v1/admin/users/{id}` and `DELETE /v1/admin/users/{id}` MUST NOT remove the last active admin from the system — a successful "demote the only admin" or "delete the only admin" call locks the deployment out of its own admin panel. The pre-fix handler did the check non-transactionally (`GetUser` + `CountActiveAdmins` + `Update`), which left a TOCTOU window where two concurrent demotions could each observe `n=2` and commit, orphaning the deployment. The fix moves the guard into the Store: `UpdateUserGuarded` and `DeleteUserGuarded` open a transaction, take `SELECT … FOR UPDATE` row locks on every active admin (so concurrent callers serialise), re-check the target row inside the transaction, and return `api.ErrLastAdmin` (mapped to HTTP `409 Conflict` with `last_admin_protection`) when the operation would empty the active-admin set. The memStore mirrors the same semantics under a single mutex so unit tests exercise the same contract.

### Audit log

`AuditMiddleware` wraps the generated router (mounted after `AuthMiddleware` in main.go so the caller is already on the request context). It records every non-GET request plus every `/v1/admin/*` read into `audit_events` — cluster-browsing GETs are deliberately not logged so the table stays bounded to meaningful events. Password / token / OIDC secret fields in captured JSON bodies are scrubbed before insert. Insertion failures are logged at ERROR but never surface as a 5xx: losing the CMDB because the audit table is briefly unreachable would be a worse outcome than a gap in the log. `GET /v1/admin/audit` returns a paginated, newest-first list; requires the `audit` scope so both `admin` and `auditor` roles can read it. The UI surfaces it at `/ui/admin/audit` — auditors see only that tab, admins see it alongside Users / Tokens / Sessions.
- `internal/api/` — generated server (`api.gen.go`), hand-written handlers (`server.go`, `auth_handlers.go`), `Store` interface (`store.go`) with `ErrNotFound` / `ErrConflict` sentinels. RFC 7807 `application/problem+json` for all errors. `auth.go` is a thin adapter wrapping `auth.Middleware` into `api.MiddlewareFunc`.
- `internal/auth/` — password hashing (argon2id, PHC-encoded), token minting (`argos_pat_<8hex>_<32urlb64>`), session cookie helpers, OIDC provider wiring (`coreos/go-oidc/v3` + `golang.org/x/oauth2`, PKCE/state/nonce primitives, issuer discovery, ID-token verification), and `Middleware()` that resolves cookie → bearer → 401 and attaches `Caller{id, kind, role, scopes}` to the request context. Role → scope mapping lives here (`ScopesForRole`). Store interface is narrow — the PG store implements it alongside the wider `api.Store`.
- `internal/store/` — PostgreSQL implementation of `api.Store` using `pgx/v5`. Cursor-paginated list, merge-patch updates, embedded `goose` migrations. FK chain: `clusters` → `namespaces` / `nodes` / `persistent_volumes` → `pods` / `workloads` / `services` / `ingresses` / `persistent_volume_claims`, all `ON DELETE CASCADE`. Workloads are polymorphic on a `kind` discriminator per ADR-0003; kind-specific detail lives in the `spec` JSONB column. Pods and Workloads also carry a `containers` JSONB column (`{name, image, image_id?, init}` entries) for SBOM / CVE workflows. Pods also carry a nullable `workload_id` FK to their top-level controlling Workload (`ON DELETE SET NULL` — a deleted Workload doesn't cascade-delete its pods, they're reaped by the normal pod-reconcile pass). PVCs carry a nullable `bound_volume_id` FK to the bound PV with the same `ON DELETE SET NULL` semantics. Nodes carry an enriched field set for incident response: role, cloud identity (`provider_id`, `instance_type`, `zone`), networking (`internal_ip`, `external_ip`, `pod_cidr`), OS stack (`kernel_version`, `operating_system`, `container_runtime_version`, `kube_proxy_version`), capacity + allocatable quadruples for cpu/memory/pods/ephemeral_storage, `conditions` + `taints` JSONB arrays, and the `ready`/`unschedulable` flags. Ingresses and Services carry a `load_balancer` JSONB column mirroring `status.loadBalancer.ingress[]` (`{ip?, hostname?, ports?}`) so on-prem VIPs — MetalLB, Kube-VIP, hardware LBs — surface alongside cloud-provisioned ones. `ListPods` and `ListWorkloads` accept a `PodListFilter` / `WorkloadListFilter` struct with optional `ImageSubstring` (JSONB ILIKE over the containers array) and `NodeName` so the UI's image-search and node-impact views can query server-side.
- `internal/eol/` — End-of-life enricher (ADR-0012). Background goroutine that periodically queries the endoflife.date API and annotates clusters and nodes with lifecycle status (`eol`, `approaching_eol`, `supported`). Controlled at runtime via the `eol_enabled` setting in the `settings` table (toggled by admins in the UI at Admin > Settings). Env vars (`ARGOS_EOL_INTERVAL`, `ARGOS_EOL_APPROACHING_DAYS`, `ARGOS_EOL_BASE_URL`) configure the enricher behaviour; `ARGOS_EOL_ENABLED` seeds the DB setting on startup. The enricher writes structured JSON annotations under `argos.io/eol.<product>` keys on the entity's `annotations` JSONB column, preserving existing annotations via read-then-merge. Matchers cover Kubernetes version (clusters + nodes), container runtime (`containerd`, `cri-o`, `docker`), OS image (14 distros), and Linux kernel version. The annotation includes `latest_available` — the latest version of the product published on endoflife.date (newest cycle's latest patch), so operators can see how far behind they are. The UI exposes a dedicated EOL Inventory page (`/ui/eol`) with summary cards (filterable by click), a sortable table with column groups separating "what we run" (Status, Product, Version, Patch, Entity, Cluster) from "what's available" (Latest Available, EOL Date, Checked), row-level red/orange highlighting by EOL status, and status badges. The push collector is not affected — enrichment runs centrally in argosd against the store. The `settings` table is a single-row table (`id=1 CHECK`) with runtime feature toggles; `GET /v1/admin/settings` and `PATCH /v1/admin/settings` are hand-written routes (not in the OpenAPI spec) gated on admin scope.
- `internal/mcp/` — Model Context Protocol server (ADR-0014). Exposes 17 read-only CMDB tools for AI agents over SSE and stdio transports. Bearer token auth reuses the standard argosd token verification. Controlled at runtime via the `mcp_enabled` setting in the `settings` table (toggled by admins in the UI at Admin > Settings). Env vars (`ARGOS_MCP_ENABLED`, `ARGOS_MCP_TRANSPORT`, `ARGOS_MCP_ADDR`, `ARGOS_MCP_TOKEN`) configure the server; `ARGOS_MCP_ENABLED` seeds the DB setting on startup. Prometheus metrics track tool invocations and durations.
- `internal/impact/` — Impact analysis graph traverser (ADR-0013). `Traverser` walks FK relationships bidirectionally from any CMDB entity to build a dependency graph. Supports 9 entity types (cluster, node, namespace, pod, workload, service, ingress, PV, PVC) with 4 relation types (`contains`, `owns`, `hosts`, `binds`). Depth-limited (1–3 hops, default 2). `HandleImpact` serves `GET /v1/impact/{entity_type}/{id}?depth=2` — hand-written route gated on `read` scope, mounted in `main.go` alongside the settings routes. Returns a flat `Graph{Root, Nodes, Edges}` JSON. The UI renders the graph as an interactive SVG diagram (hierarchical layout, click-to-navigate) via the `ImpactSection` component embedded in every entity detail page. Prometheus metrics: `argos_impact_queries_total{entity_type}`, `argos_impact_query_duration_seconds{entity_type}`.
- `internal/metrics/` — Prometheus registry and exporter. `/metrics` endpoint is mounted on the main HTTP mux unauthenticated (Prometheus scrape convention). Exposes HTTP request/duration counters, collector upsert / reconcile / error counters and last-poll gauges per `(cluster, resource)`, plus `argos_build_info`. `InstrumentHandler` wraps the API mux to count requests by method, route pattern, and status class.
- `ui/` — Vite + React + TypeScript SPA per ADR-0006. Built bundle is embedded into the argosd binary via `//go:embed all:dist` (see `ui/embed.go`) and served at `/ui/*`; root `/` redirects to `/ui/`. Per ADR-0007, humans log in with username + password and receive a server-side session cookie; the SPA reads `/v1/auth/me` on every route change to drive role-aware rendering and forced-rotation redirects. Logout hits `/v1/auth/logout`. Build-tag `noui` disables the embed for backend-only workflows (`make build-noui`); the default `make build` requires `make ui-build` to have produced `ui/dist/` first. Dev loop: `make ui-dev` serves the SPA on :5173 with `/v1` + `/healthz` + `/metrics` proxied to a local argosd on :8080 for hot reload. The UI uses the Argos Design System: all CSS values reference canonical token variables (~50 custom properties in `:root`), no hardcoded hex values outside token definitions. Typography: Space Grotesk (headings/body) + JetBrains Mono (code) via Google Fonts. Layout: collapsible left sidebar navigation with 11 inline SVG entity icons (`ui/src/icons.tsx`), top header bar for app title and user controls.
- `internal/collector/` — Kubernetes polling collector. Each tick fetches the API server version (refreshing the cluster record), lists nodes (upsert by `(cluster_id, name)`; reads the full enriched field set from `status.nodeInfo` + `status.capacity` + `status.allocatable` + `status.addresses` + `status.conditions` + `spec.providerID` + `spec.podCIDR` + `spec.taints` + `spec.unschedulable`, derives role from `node-role.kubernetes.io/*` labels, and reads instance-type / zone from the well-known topology labels), lists cluster-scoped PersistentVolumes (upsert by `(cluster_id, name)`, returns name → id map for PVC linking), lists namespaces (upsert by `(cluster_id, name)`, returns name → id map), lists workloads via three `AppsV1` list calls folded into one `[]WorkloadInfo` tagged by `Kind` (upsert by `(namespace_id, kind, name)`, ingested before pods so each pod's `workload_id` can be resolved), lists pods cluster-wide (upsert by `(namespace_id, name)`; the pod's controlling `ownerReference` is walked — `ReplicaSet` → `Deployment` via a side `ListReplicaSetOwners` call, or direct for `StatefulSet` / `DaemonSet` — to set `workload_id`; unmodelled kinds like `Job` leave the FK null), lists services cluster-wide (upsert by `(namespace_id, name)`; flattens `status.loadBalancer.ingress[]` into the `load_balancer` JSONB), lists ingresses via `NetworkingV1` (upsert by `(namespace_id, name)`, rules/tls/load_balancer flattened into JSONB), and lists PersistentVolumeClaims cluster-wide (upsert by `(namespace_id, name)`; each PVC's `spec.volumeName` is resolved against the PV map to set `bound_volume_id`, or null when pending / the PV isn't listed this tick). When `ARGOS_COLLECTOR_RECONCILE=true` (default), rows that disappeared from the live listing are deleted so the CMDB mirrors the cluster — required for ANSSI cartography fidelity. Pods, workloads, services, ingresses, and PVCs reconcile per-namespace; nodes and PVs reconcile cluster-scoped; workload reconcile keys on the `(kind, name)` tuple so a deleted Deployment `web` doesn't wipe the still-live StatefulSet `web`. Reconciliation only runs after a successful list, so a transient Kubernetes error never wipes the store. Disabled by default; enable with `ARGOS_COLLECTOR_ENABLED=true`. Multi-cluster topology per ADR-0005: `ARGOS_COLLECTOR_CLUSTERS` is a JSON array of `{name, kubeconfig}` tuples, one collector goroutine per entry, all sharing the store; the legacy `ARGOS_CLUSTER_NAME` + `ARGOS_KUBECONFIG` still work as a single-cluster shortcut. An empty `kubeconfig` falls back to in-cluster config.
- `internal/httputil/` — shared request-introspection helpers that gate trust in attacker-controllable proxy headers (ADR-0017 §2). `ParseTrustedProxies` parses `ARGOS_TRUSTED_PROXIES` (comma-separated CIDR list) into `[]*net.IPNet`; empty input → empty slice (the secure default). `ClientIP(r, trusted)` returns the source IP for rate-limiting and audit: when the immediate peer (`r.RemoteAddr`) is not in the trust list, X-Forwarded-For is ignored entirely; when it is trusted, XFF is walked **right-to-left** through trusted hops and the first untrusted IP is returned — attackers can prepend XFF values but cannot append past trusted hops, so prepended values are unreachable. `IsHTTPS(r, trusted)` reports whether the request actually arrived over TLS, either natively (`r.TLS != nil`) or via a TLS-terminating peer in the trust list that set `X-Forwarded-Proto: https`; with an empty trust list, XFP is ignored. Used by the rate limiter (`auth_handlers.go`), audit middleware (`audit.go`), session secure-cookie decision (`internal/auth/session.go`), and trust-aware HSTS (`security_headers.go`).
- `internal/secrets/` — AES-256-GCM envelope encryption for long-lived secrets stored in the CMDB (ADR-0015 §4 / IMP-002). Master key from `ARGOS_SECRETS_MASTER_KEY` (base64-encoded 32 bytes; rejected at startup on any other length). `Encrypter.Encrypt` requires AAD bound to the row's primary key (the `cloud_account` UUID's bytes) so a backup-restore cannot move a ciphertext between rows. Persisted form is `{Bytes, Nonce, KID}` — the `kid` column lets a future rotation ADR introduce a multi-key registry without a schema change; only `v1` is recognised today. `Fingerprint()` returns the first 8 hex chars of SHA-256(masterKey) and is logged at startup so operators can confirm the right key is loaded without exposing the key itself. Master key is required only when at least one `cloud_accounts` row carries a non-NULL `secret_key_encrypted`; argosd refuses to start otherwise.
- `internal/ingestgw/` — DMZ ingest gateway package (ADR-0016). Implements `argos-ingest-gw`, the stateless reverse-proxy binary deployed in a perimeter network. Key sub-units: `allowlist.go` (hardcoded 18-route `(method, path)` write-only table — the literal security boundary auditors read during a review), `cache.go` (bounded LRU keyed on `sha256(token)` with independent positive/negative TTLs and single-flight dedupe for thundering-herd on token rotation), `verify_client.go` (`VerifyClient` calls argosd's `POST /v1/auth/verify` over the shared mTLS `*http.Client`; returns `ErrVerifyDenied` on 401), `tls_reload.go` (`CertReloader` uses `fsnotify` for hot cert rotation without a pod restart — works across Vault Agent atomic-rename, cert-manager, and manual file writes), `proxy.go` (request forwarding with header strip, body cap, and `X-Argos-Verified-*` injection defence), `server.go` (wires everything into a single `http.Handler`; `Config.Check()` validates required fields), `metrics.go` (private Prometheus registry with `Outcome*` constants, 12 counters/histograms/gauges). The `Route` table in `allowlist.go` and `IngestRoutes` in `internal/api/ingest_mux.go` must be kept in sync — a route argosd serves but the gateway blocks is acceptable defence in depth; the reverse is a configuration error that 404s at the listener.
- `internal/vmcollector/` — VM-collector polling loop (ADR-0015 §IMP-003), shared between the standalone `argos-vm-collector` binary and the unit tests. `Collector.Run` ticks every `Interval`: ensure credentials (with cached refresh on `CredentialRefresh` cadence, default 1 h), build a `provider.Provider` via the `ProviderFactory`, list VMs, apply the local pre-filter (`internal/vmcollector/filter`), upsert each via the apiclient (skipping `ErrAlreadyKubeNode` 409s with a log line), reconcile tombstones, then `PATCH /status` with `last_seen_at` / `last_error`. Sub-packages: `provider/` (the cloud-provider seam — `Provider` interface, `VM` canonical struct, `Outscale` impl wrapping `osc-sdk-go/v2`, `fake.go` test fake, plus `CanonicalPowerState`, `DeriveRegionFromZone`, `ParseInstanceTypeCapacity` helpers); `apiclient/` (HTTP-backed `Store` mirroring `internal/collector/apiclient`'s transport setup — CA / mTLS / extra-headers / proxy support — but exposing only the five collector-side methods, with `ErrAlreadyKubeNode` / `ErrNotRegistered` / `ErrAccountDisabled` sentinels and 5xx retry-with-exponential-backoff capped at 3 attempts); `filter/` (cheap pre-filter dropping VMs tagged `OscK8sClusterID/*`, `OscK8sNodeName=*`, or `argos.io/ignore=true` — server-side dedup against `nodes.provider_id` is the canonical check, this just saves the round-trip per kube worker). `metrics.go` exposes a private `prometheus.Registry` separate from argosd's so the two binaries can run side by side; counters/gauges cover `ticks_total{status}`, `tick_duration_seconds`, `vms_observed`, `vms_skipped_kubernetes_total`, `credential_refreshes_total{result}`, `last_success_timestamp_seconds`, `build_info{version}`.
- `cmd/argos-ingest-gw/` — standalone DMZ ingest gateway binary (ADR-0016). Stateless, distroless base, UID 65532, env-var configured via `ARGOS_INGEST_GW_*`. One instance per DMZ. Exposes an inbound TLS listener (`:8443`, fronted by Envoy/WAF) and a health/metrics listener (`:9090`, no TLS, pod-IP only). No database, no queue, no replay buffer. Builds `CGO_ENABLED=0` from `Dockerfile.ingest-gw` via a two-stage build (no Node, no UI embed).
- `cmd/argos-vm-collector/` — standalone push-mode binary (ADR-0015 §1, §IMP-004). Stateless: no DB, no listener, env-var configured (`ARGOS_SERVER_URL`, `ARGOS_API_TOKEN`, `ARGOS_VM_COLLECTOR_PROVIDER` / `ACCOUNT_NAME` / `REGION` / `INTERVAL` / `FETCH_TIMEOUT` / `RECONCILE` / `CREDENTIAL_REFRESH`, plus the standard `ARGOS_CA_CERT` / `ARGOS_CLIENT_CERT` / `ARGOS_CLIENT_KEY` / `ARGOS_EXTRA_HEADERS` mTLS+gateway block reused from `argos-collector`). Built `CGO_ENABLED=0`, distroless base, UID 65532. One binary instance per cloud account; multi-account = N deployments. Has no AK/SK env var of its own — credentials live exclusively in argosd's `cloud_accounts` table and the collector fetches them on boot (and every `CREDENTIAL_REFRESH` interval) via `GET /v1/cloud-accounts/by-name/{name}/credentials` with its bound vm-collector PAT.
- `cloud_accounts` table (ADR-0015 §3) — operator-editable cloud-provider accounts. Columns: identity (`id`, `provider`, `name` — UNIQUE per provider, `region`), credentials (`access_key` plaintext + `secret_key_encrypted` BYTEA + `secret_key_nonce` + `secret_key_kid`; AK plaintext because it's a public identifier in request signatures, SK encrypted with AES-256-GCM AAD-bound to `id`), heartbeat (`last_seen_at`, `last_error`, `last_error_at`), curated metadata (`owner`, `criticality`, `notes`, `runbook_url`, `annotations`), lifecycle (`created_at`, `updated_at`, `disabled_at`), and a `status` discriminator with four values: `pending_credentials` (collector registered, admin hasn't set AK/SK yet — surfaces as a red banner on the admin home), `active` (collector ticking successfully), `error` (last tick reported a failure — the collector itself flips back to `active` on the next success), `disabled` (admin disabled — credentials-fetch returns 403). The hybrid-onboarding flow (ADR-0015 §6): the collector boots → `GET /credentials` 404s → collector retries with `POST /v1/cloud-accounts` to register a placeholder row → admin pastes AK/SK in the UI → collector picks them up on the next refresh tick. Admin can rotate AK/SK at any time without redeploying the collector (PATCH `/credentials` re-encrypts; collector picks up the new SK within `CREDENTIAL_REFRESH`).
- `virtual_machines` table (ADR-0015 §2) — non-Kubernetes platform VMs (VPN, DNS, Bastion, Vault, …). Top-level table FK to `cloud_accounts(id)` `ON DELETE CASCADE`; intentionally separate from `nodes` so K8s semantics stay clean. Schema mirrors the enriched `nodes` shape where it overlaps (identity, networking, capacity quadruple, curated metadata) and adds the cloud-native columns the Outscale API exposes: `image_id` (AMI), `keypair_name`, `boot_mode`, `vpc_id`, `subnet_id`, `nics` JSONB, `security_groups` JSONB, `block_devices` JSONB, `root_device_type`, `root_device_name`, `provider_creation_date`, `deletion_protection`. Drops the K8s-specific columns (`kubelet_version`, `kube_proxy_version`, `container_runtime_version`, `pod_cidr`, `conditions`, `taints`, `unschedulable`, `allocatable_*`). Soft-delete via `terminated_at`: `POST /v1/virtual-machines/reconcile` flips `terminated_at = NOW()` + `power_state = 'terminated'` + `ready = false` for rows whose `provider_vm_id` is not in the keep list; rows are never hard-deleted by reconciliation so the SecNumCloud audit history of decommissioned VMs is preserved (retention policy is a follow-up). A re-appearing `(cloud_account_id, provider_vm_id)` is resurrected by clearing `terminated_at`.
- `vm-collector` token scope (ADR-0015 §5 / IMP-008) — narrow scope reserved for collector PATs. Granted endpoints: `GET /v1/cloud-accounts/by-name/{name}/credentials` (and the by-id variant), `POST /v1/cloud-accounts` (idempotent first-contact registration), `PATCH /v1/cloud-accounts/{id}/status` (heartbeat-only — cannot modify name, AK, SK, or curated metadata), `POST /v1/virtual-machines`, `POST /v1/virtual-machines/reconcile`. PATs are bound to a single `cloud_account_id` at issuance — the `tokens.bound_cloud_account_id` column (added in `00025_*`) is enforced by `auth.Caller.EnforceCloudAccountBinding`; a vm-collector PAT bound to account A cannot read account B. **`auth.HasScope` deliberately does NOT let the admin scope imply `vm-collector`** — this preserves the SK-is-write-only-from-admin-endpoints guarantee: only collector tokens can read plaintext SK. The token-issuance UI requires picking a bound cloud account when the `vm-collector` preset is selected.
- Credentials-fetch endpoint — `GET /v1/cloud-accounts/by-name/{name}/credentials` (and `/by-id`) is the only place plaintext SK leaves the database. Returns 404 (`cloud_account_not_registered`) when the row doesn't exist, 403 when `disabled_at IS NOT NULL` or `status = 'pending_credentials'`, 200 with `{access_key, secret_key, region, provider}` over TLS otherwise. Audit-logged on every call (caller id, account name, timestamp); the audit middleware special-cases this path to log the call without ever capturing the response body, even at debug level.
- Public-listener TLS posture and proxy trust (ADR-0017) — closes AUTH-VULN-01/02/03/04 from the 2026-04-28 pentest. Four env vars on argosd's main listener: `ARGOS_PUBLIC_LISTEN_TLS_CERT` + `ARGOS_PUBLIC_LISTEN_TLS_KEY` opt argosd into native TLS 1.3 with hot cert reload (mtime-driven via `newCertReloader` in `cmd/argosd/main.go` — simpler than the gateway's fsnotify variant because rotations are infrequent on the trusted-zone listener); `ARGOS_TRUSTED_PROXIES` (comma-separated CIDR list, empty by default) defines the immediate-peer set whose X-Forwarded-For / X-Forwarded-Proto headers argosd will honor — empty = ignore both headers entirely (secure default); `ARGOS_REQUIRE_HTTPS=true` is the §3 startup guard that refuses boot unless either native TLS is configured **or** a trusted-proxy + always-secure-cookie posture is in place (`trustedProxies non-empty AND cookiePolicy=SecureAlways`), so a misconfigured operator cannot accidentally ship credentials over plaintext. The trust list is wired into `auth.Middleware` (rate-limit IP), `AuditMiddleware` (logged source IP), `setSecurityHeaders` (HSTS only emitted on TLS or trusted-XFP requests, with `forceEmit=true` available when the operator knows every entrypoint is HTTPS), and `internal/auth/session.go` (`secureFlag` consults `httputil.IsHTTPS` instead of `r.TLS != nil` so `Secure` cookies survive a TLS-terminating reverse proxy). Helm chart wiring lives under `charts/argos/values.yaml` `argosd.tls.existingSecret` (cert/key from a Secret reference) and `argosd.trustedProxies` / `argosd.requireHTTPS`.
- DMZ ingest gateway (ADR-0016) — argosd grows a second mTLS-only listener (`:8443`, opt-in via `ARGOS_INGEST_LISTEN_ADDR`; when unset, the listener never starts and existing deployments are unaffected). The listener is wired by `api.NewIngestMux` and registers exactly 19 routes: the 18 writes the K8s push collector uses plus `POST /v1/auth/verify`. Auth and audit middleware apply as on the public listener; `AuditMiddleware` receives `source="ingest_gw"` so the `audit_events.source` column distinguishes DMZ-origin writes (`ingest_gw`) from public-listener calls (`api`) and synthetic events (`system`). The argosd-side env vars are `ARGOS_INGEST_LISTEN_ADDR`, `ARGOS_INGEST_LISTEN_TLS_CERT`, `ARGOS_INGEST_LISTEN_TLS_KEY`, `ARGOS_INGEST_LISTEN_CLIENT_CA_FILE`, and `ARGOS_INGEST_LISTEN_CLIENT_CN_ALLOW`.
- `POST /v1/auth/verify` (ADR-0016 §5) — internal-only endpoint registered exclusively on the mTLS-only ingest listener; not reachable on `:8080`. The DMZ gateway calls it to short-circuit invalid tokens before forwarding writes into the trusted zone. Authentication for the call itself is the listener-level mTLS client cert; the `token` field in the JSON body is the PAT to be verified, not the gateway's identity. Rate-limited at 100 req/s per source IP (burst 200) via `api.VerifyRateLimiter`; pass nil to the limiter in tests to disable. Response body carries `valid`, `caller_id`, `kind`, `scopes`, and `bound_cloud_account_id`; the `token` field is scrubbed from audit logs by the existing body-scrubber.
- `POST /v1/clusters` is now idempotent on `name` (ADR-0016 §6) — `api.Store.EnsureCluster` replaces `CreateCluster` (which returned 409 on a duplicate name). A new row returns `201 Created`; an existing row returns `200 OK` with the existing record (request body ignored on hit). Callers wanting to update fields on an existing cluster still use `PATCH /v1/clusters/{id}`. `EnsureCluster` never returns `ErrConflict`; concurrent inserts are serialised at the DB via `INSERT … ON CONFLICT DO NOTHING`. The collector no longer needs a startup `GET /v1/clusters?name=…` — it unconditionally POSTs and follows up with PATCH regardless of 200 or 201.
- Audit middleware extension (ADR-0015 §IMP-007) — `AuditMiddleware` now wraps the hand-written cloud-accounts and VM routes too (previously a security gap: those routes were mounted alongside settings/impact and bypassed the codegen mux). `shouldAudit` covers `GET /v1/cloud-accounts/.../credentials` so credential reads land in the log; the existing scrubber list is extended with `secret_key` and `access_key` so any stray captured request body is sanitised before insert.
- VM dedup logic — server-side check on `POST /v1/virtual-machines`: `SELECT id FROM nodes WHERE provider_id LIKE '%' || $1 || '%'` against the posted `provider_vm_id`. Underscore (`_`) and percent (`%`) inside `provider_vm_id` are escaped before interpolation so a maliciously-named VM cannot match every node row. Match → 409 `{"error":"already_inventoried_as_kubernetes_node","node_id":"<uuid>"}`; the collector logs and continues. Tag-independent: works regardless of which cloud-controller-manager is in use, because Outscale's CCM stamps the `VmId` substring (`i-96fff41b`) into `node.spec.providerID` (`aws:///<az>/<vmid>`).
- `migrations/` — timestamped SQL migrations, embedded in the binary via `migrations/embed.go`.
- `.github/workflows/ci.yml` — GitHub Actions pipeline: `setup-node@v4` (Node 22, npm-cache keyed on `ui/package-lock.json`) + `npm ci && npm run build` to produce `ui/dist` so downstream `go build` can `//go:embed` it, then codegen-drift check, `go vet`, `go build`, `go test -race` against a Postgres service container (integration tests gated on `PGX_TEST_DATABASE` run in CI), `golangci-lint`, and a parallel `docker build` job that verifies the image compiles (no publish yet).
- `Dockerfile` — three-stage build: `node:22-alpine` UI builder → `golang` builder (embeds `ui/dist` via `//go:embed`) → `gcr.io/distroless/static-debian12:nonroot` runtime. Produces a static (`CGO_ENABLED=0`) `argosd` binary, runs as UID 65532. Override `VERSION` build arg to stamp `main.version`. Local build via `make docker-build` (tags `argos:dev`).
- `deploy/` — reference Kustomize manifests for running `argosd` in a Kubernetes cluster cataloguing itself via in-cluster ServiceAccount (namespace, ServiceAccount + ClusterRole with `list` on the catalogued kinds — nodes, namespaces, pods, services, persistentvolumes, persistentvolumeclaims in the core group; deployments, statefulsets, daemonsets, replicasets in `apps`; ingresses in `networking.k8s.io`; plus Deployment, ClusterIP Service, Secret template). `deploy/README.md` covers the full walkthrough including the multi-cluster variant (mount multiple kubeconfigs, switch to `ARGOS_COLLECTOR_CLUSTERS`).
- `scripts/seed-demo.sh` — populates a running argosd (on `:8080`, token `dev` by default) with a realistic multi-cluster inventory for demos / screenshots / UI development. Re-runnable after a `TRUNCATE clusters CASCADE`.

Cluster carries curated-metadata columns (`owner`, `criticality`, `notes`, `runbook_url`, `annotations` JSONB) so operators can record "who owns this cluster / how critical is it / where's the runbook" on top of the collector-populated facts. The columns are merge-patched through the existing `PATCH /v1/clusters/{id}`; editor/admin can write, viewer/auditor see read-only. The collector never touches them — its per-tick update sets only `kubernetes_version`, and `UpdateCluster`'s merge semantics leave unset fields alone. UI surfaces them as an "Ownership & context" card on `/ui/clusters/:id` with an inline Edit flip. Follow-up work to propagate the same pattern to namespaces / nodes / workloads is queued. Snapshots / time-travel (per the original ADR-0001 roadmap) remain a longer-horizon item.
